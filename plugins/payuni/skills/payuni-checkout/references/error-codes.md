# PAYUNi 錯誤代碼

## 交易錯誤

| 代碼 | 說明 | 解決方式 |
|------|------|---------|
| TRA10001 | 查無此筆交易 | 確認訂單編號正確 |
| TRA10002 | 驗證失敗 | 確認加密參數與 HashKey/HashIV 正確 |
| TRA10003 | 商店代號錯誤 | 確認 MerchantID 正確 |
| TRA10004 | 金額格式錯誤 | 金額必須為正整數 |
| TRA10005 | 訂單編號重複 | 使用不同的訂單編號 |
| TRA10006 | 參數格式錯誤 | 確認參數格式符合規範 |

## 付款錯誤

| 代碼 | 說明 | 解決方式 |
|------|------|---------|
| PAY20001 | 信用卡授權失敗 | 請用戶確認卡片資訊或更換卡片 |
| PAY20002 | 信用卡餘額不足 | 請用戶確認額度或更換付款方式 |
| PAY20003 | 交易超時 | 請用戶重新發起付款 |
| PAY20004 | 付款方式不支援 | 確認商店有開通該付款方式 |

## Webhook 錯誤

| 代碼 | 說明 | 解決方式 |
|------|------|---------|
| WHK30001 | CheckCode 驗證失敗 | 確認 HashKey/HashIV 正確，參數順序正確 |
| WHK30002 | 重複的 Webhook | 該交易已處理過，可忽略 |
| WHK30003 | 訂單不存在 | 確認訂單編號對應的訂單存在 |

## 系統錯誤

| 代碼 | 說明 | 解決方式 |
|------|------|---------|
| SYS90001 | 系統維護中 | 稍後再試 |
| SYS90002 | API 連線失敗 | 確認網路連線正常 |
| SYS90003 | 請求超時 | 稍後再試或聯繫客服 |
